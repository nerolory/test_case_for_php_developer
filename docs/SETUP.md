# Инструкция по развёртыванию проекта

Регламент развертывания REST API «Список задач» внутри изолированного Docker-контейнера.

## Системные требования
* Установленный и запущенный демон **Docker** с поддержкой инструмента **Docker Compose**.
* Доступность команд `docker` и `docker compose` в терминале хост-машины.
* **Критическое условие:** Свободные сетевые порты **8080** (HTTP) и **3306** (MySQL) на хост-машине. Убедитесь, что данные порты не заняты другими локальными сервисами перед началом сборки.
* *Примечание: Дополнительное системное окружение (PHP 8.3, Composer, СУБД) разворачивается автоматически внутри контейнеров.*

## Состав окружения
| Сервис | Контейнер | Назначение |
| --- | --- | --- |
| **app** | `tasks_api_app` | PHP 8.3-FPM, Laravel, расширение `redis` |
| **nginx** | `tasks_api_nginx` | HTTP-сервер, проброс внешнего порта **8080** |
| **mysql** | `tasks_api_mysql` | СУБД MySQL 8.4, выделенная база данных `tasks` |
| **redis** | `tasks_api_redis` | Redis 7, хранилище кэша списков и сущностей |

## Пошаговый запуск

### 1. Клонирование репозитория и переход в каталог
```bash
git clone https://github.com/nerolory/test_case_for_php_developer.git
cd test_case_for_php_developer
```

### 2. Инициализация конфигурации окружения
```bash
cp .env.example .env
```
*Конфигурация по умолчанию полностью адаптирована под локальную Docker-сеть.*

### 3. Сборка образов и запуск контейнеров
```bash
docker compose up -d --build
```
*Проверка текущего статуса запущенных контейнеров:*
```bash
docker compose ps
```
*Все запущенные сервисы должны перейти в статус `running`, а контейнер `mysql` — в состояние `healthy`.*

### 4. Внутренняя инициализация приложения
Выполняется строго внутри запущенного контейнера `app`:
```bash
docker compose exec app php artisan key:generate
docker compose exec app php artisan migrate --seed
```
*Полный перезапуск и очистка БД с повторным наполнением (при необходимости):*
```bash
docker compose exec app php artisan migrate:fresh --seed
```

## Доступные сетевые интерфейсы
* **Интерфейс API:** http://localhost:8080/api/tasks
* **Интерактивная документация Swagger UI:** http://localhost:8080/api/documentation

*Принудительное обновление спецификации Swagger:*
```bash
docker compose exec app php artisan l5-swagger:generate
```

## Регламент повседневной разработки
| Операция | Команда |
| --- | --- |
| Приостановка работы сервисов | `docker compose stop` |
| Возобновление работы сервисов | `docker compose start` |
| Удаление контейнеров (с сохранением томов БД) | `docker compose down` |
| Полное уничтожение контейнеров и томов БД | `docker compose down -v` |
| Просмотр логов всех сервисов | `docker compose logs -f` |
| Интерактивный терминал контейнера разработки | `docker compose exec app sh` |

## Диагностика неисправностей (Troubleshooting)

* **Порт 8080 занят сторонним процессом:** Смените внешний порт в конфигурации `docker-compose.yml` (например, `"8081:80"`) и синхронно обновите параметры `APP_URL` и `L5_SWAGGER_CONST_HOST` в локальном файле `.env`.
* **Сбой подключения при миграциях (БД не готова):** Контейнер `mysql` инициализируется дольше остальных. Дождитесь перевода статуса контейнера в `healthy` через команду `docker compose ps` и повторите миграции.
* **Нарушение прав на запись в каталогах Laravel:** Контейнер автоматически перевыставляет права пользователя `www-data` через встроенную точку входа (`entrypoint`). Выполните принудительную пересборку: `docker compose up -d --build`.
* **Изменения в кодовой базе не вступают в силу:** Каталог монтируется как сквозной `volume`. При обновлении зависимостей внутри `composer.json` выполните: `docker compose exec app composer install`.
